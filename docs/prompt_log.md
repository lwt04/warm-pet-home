# AI 使用记录

以下记录用于课程提交留痕，便于说明本项目在开发过程中确实使用了 AI 辅助。

## 1. 项目选题与功能梳理

- 时间：2026-07-14
- 目标：确定适合 HBuilder X / uni-app 的项目主题
- AI 输出要点：建议选择“流浪动物救助与领养”作为主线，页面包含领养页、宠物圈、注意事项、个人中心，并补充登录、发布、申请、收藏、点赞等功能
- 对应成果：完成项目主题确定、页面结构初稿和功能清单

## 2. 项目骨架与 Tab 设计

- 时间：2026-07-14
- 目标：搭建 uni-app 项目骨架和底部导航
- AI 输出要点：按领养、宠物圈、注意事项、个人中心 4 个 Tab 组织页面，并预留图标路径
- 对应成果：完成项目骨架、`pages.json` 配置和 `static/tabbar/` 图标规范

## 3. 后端接口设计

- 时间：2026-07-16
- 目标：为前端联动设计本地后端
- AI 输出要点：使用 Node.js + Express + SQLite，接口覆盖登录注册、宠物发布、申请审核、收藏、动态、点赞、评论
- 对应成果：完成 `server/index.js` 和 `server/db.js`，提供可运行的本地 API

## 4. 头像与图片持久化

- 时间：2026-07-18
- 目标：解决图片刷新后空白的问题
- AI 输出要点：统一把图片路径规范成可访问地址，避免把 `blob:` 临时路径当作长期数据保存
- 对应成果：新增 `common/image.js`，修复宠物、动态、头像图片显示

## 5. 交互细节优化

- 时间：2026-07-17 至 2026-07-18
- 目标：优化发布后跳转、申请后返回、消息红点、收藏点赞跳转等体验
- AI 输出要点：发布成功后自动进入详情页，申请成功后返回详情页，消息中心显示未读数量
- 对应成果：页面交互更完整，演示链路更顺畅
