# AI Code Review 记录

以下为基于 AI 辅助检查得到的优化建议与处理结果，供课程提交使用。

## 检查范围

- `server/index.js`
- `server/db.js`
- `common/api.js`
- `common/image.js`
- `pages/pet/publish.vue`
- `pages/post/publish.vue`
- `pages/user/edit.vue`

## 发现的问题与处理

### 1. 图片临时地址不适合长期保存

- 问题：前端直接保存 `blob:` 或临时路径，刷新后无法稳定显示
- 处理：新增统一图片路径规范方法，并把 `blob:` 只作为临时选择结果处理
- 结果：宠物详情、宠物圈、头像显示稳定

### 2. 后端读取旧数据时存在兼容性风险

- 问题：数据库里历史图片字段可能混有绝对路径、临时路径和项目路径
- 处理：后端返回前统一规范化，旧数据也做了一次批量修复
- 结果：旧内容与新内容都能兼容显示

### 3. 发布和审核的交互可以更顺滑

- 问题：发布成功后还要手动返回，体验不够顺
- 处理：发布后自动跳转详情页，申请后自动回到详情页
- 结果：演示时流程更自然

### 4. 文档缺少提交材料入口

- 问题：README 只有功能介绍，缺少提交材料说明
- 处理：补充了 API 文档、开发进度和后续计划说明
- 结果：仓库更容易给老师检查

## 总体结论

项目整体结构清晰，接口数量和页面数量都满足课程要求。主要风险不在功能本身，而在于后续提交材料是否完整、线上部署是否稳定。
