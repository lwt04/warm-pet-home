# 个人实训总结

本次实训我完成了一个流浪动物救助与领养平台“暖宠归家”，整体体验是从零搭建一个能跑、能看、能演示的项目，比单纯写页面要复杂得多。

刚开始时，我对 HBuilder X、uni-app 和本地后端联动并没有完整概念，只是想先把页面搭出来。后来随着功能不断增加，我逐步把项目拆成几个部分：领养首页、宠物圈、注意事项、个人中心、登录注册、宠物发布、领养申请、消息提醒等。这样梳理之后，项目的逻辑终于清楚了，也更符合一个真实应用的样子。

我遇到的最大困难是图片和数据的持久化问题。前期我把 `uni.chooseImage` 得到的临时路径直接存进数据库，结果页面刷新后图片经常消失，宠物详情页和宠物圈会变成空白。这让我意识到，浏览器里的临时路径不能当成长期数据保存。后来我统一了图片路径处理规则，并把历史数据做了修复，才解决这个问题。这个过程让我对“前端显示”和“后端存储”之间的区别有了更深的理解。

另一个收获是工程化意识。课程考核并不只是看功能有没有做出来，还要看 Git 提交、README、接口文档、AI 使用记录和总结报告。这让我明白，一个项目不是只有代码本身，交付材料同样重要。为了满足老师要求，我把开发过程拆成多次提交，并补充了 API 文档、AI 留痕和审查记录。

在 AI 辅助编程方面，我的感受是它很适合做“搭架子”和“排查思路”，比如帮我整理功能顺序、设计页面结构、检查接口联动、优化文档内容。但真正落地时，还要自己理解业务逻辑，特别是登录状态、申请审核、状态流转和图片显示这些细节，不能只靠复制粘贴。

总体来说，这次实训让我从“会写页面”进步到“会组织一个完整项目”。我不仅完成了功能，也更加理解了工程化开发、数据流和项目交付的要求。
