# 暖宠归家

流浪动物救助与领养平台，使用 HBuilder X / uni-app 开发，后端使用 Node.js、Express 和 SQLite。

## 当前阶段

项目已完成前端主要页面、底部 Tab、登录注册、个人中心、领养首页、宠物详情、宠物圈、申请审核和本地后端接口联动。当前后端运行地址为：

```text
http://localhost:3000
```

前端默认进入未登录状态。用户登录后才可以发布救助、发布动态、收藏、点赞、评论和提交领养申请。

## 已完成功能

- 4 个底部 Tab：首页领养、宠物圈、注意事项、个人中心
- 登录、注册、退出登录、修改个人资料、更换头像
- 未登录个人中心提示登录/注册，已登录展示资料和功能入口
- 领养首页：搜索、分类筛选、轮播图、宠物卡片、状态展示
- 发布救助：完整表单校验，支持 1-9 张宠物图片
- 宠物详情：展示详情信息，顶部多图轮播，收藏/取消收藏，申请领养
- 申请规则：不能申请自己发布的宠物，已领养宠物不能申请
- 申请审核：发布人可以同意或拒绝；同意后宠物状态自动变为“已领养”
- 我的发布：查看自己发布的救助信息和宠物圈动态，可进入详情或删除
- 宠物圈：动态列表、点赞、收藏、评论、删除自己的评论
- 发布动态：支持 1-9 张图片，宠物圈按单图/九宫格样式展示
- 个人中心：我的申请、收到申请、我的收藏、我的点赞、消息通知红点提醒
- 后端 API：用户、宠物、申请、收藏、动态、点赞、评论等接口
- API 文档：[docs/api.md](docs/api.md)

## 后端启动

在项目根目录运行：

```bash
npm.cmd run server
```

启动后访问：

```text
http://localhost:3000/api/health
```

看到 `{"status":"ok","name":"warm-pet-home-server"}` 表示后端正常。

## 前端运行

1. 先启动后端：

```bash
npm.cmd run server
```

2. 再在 HBuilder X 中运行前端：

```text
运行 -> 运行到浏览器
```

如果前端提示“后端服务未连接”，请先确认 `http://localhost:3000/api/health` 可以正常打开。

## 部署配置

仓库已提供 `render.yaml`，也可以在 Render 中手动创建 Web Service。

部署时使用：

- 启动命令：`npm start`
- 健康检查：`/api/health`
- 数据目录：`server/data`

当前线上服务使用 Render Free 实例，长时间无人访问后首次打开可能需要等待几十秒。若升级到付费实例并使用持久化磁盘，建议把挂载路径保持为 `/opt/render/project/src/server/data`，这样 SQLite 数据不会因为重启而丢失。
